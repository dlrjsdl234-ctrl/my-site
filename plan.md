# 화면 인식(beta) 레벨업 인식 개선 계획

## 목표
- `85% 이상`, `35%p 하락`, `65% 이하` 같은 게임 속도에 직접 묶인 고정 조건을 제거한다.
- 저레벨 고속 성장과 고레벨 저속 성장을 같은 로직으로 처리한다.
- 레벨업은 “최근 정상 상승 추세 대비 유의미하게 낮은 값이 연속 관측되는 경우”로 판단한다.

## 현재 문제
- 기존 레벨업 조건은 특정 퍼센트 구간을 전제로 했다.
- 저레벨에서는 레벨업 직후 이미 높은 퍼센트까지 차거나, 85%에 도달하기 전에 다음 낮은 값이 관측될 수 있다.
- 고레벨에서는 작은 퍼센트 변화도 의미가 있으므로 단일 하락폭 기준으로는 민감도를 맞추기 어렵다.

## 개선 방향
- 정상 반영된 최근 샘플을 별도 버퍼에 저장한다.
- 최근 샘플로 현재 상승 속도와 인식 노이즈를 계산한다.
- 새 샘플이 최근 추세의 예상값보다 노이즈 기준 이상 낮아지면 레벨업 후보로 본다.
- 후보는 즉시 확정하지 않고, 낮은 값이 2회 이상 비슷하게 유지될 때 확정한다.
- 후보 이후 이전 정상값 근처로 복귀하면 오인식으로 보고 취소한다.

## 구현 내용

### 1. 정상 샘플 버퍼 추가
- 정상 증가 또는 레벨업 확정 후의 퍼센트를 `screenAcceptedRatioSamples`에 저장한다.
- 최근 15초, 최대 12개 샘플만 유지한다.

```js
let screenAcceptedRatioSamples = [];
```

### 2. 최근 추세 계산
- `getRecentRatioTrend()`에서 최근 정상 샘플을 기반으로 다음 값을 계산한다.
  - `velocity`: 최근 정상 상승 속도
  - `noise`: 실제 변화와 예상 변화의 차이를 기반으로 한 동적 노이즈
  - `expectedRatio`: 현재 시점에서 예상되는 퍼센트
  - `positiveSteps`: 최근 정상 상승 횟수

### 3. 동적 레벨업 후보 판정
- 기존 고정 조건을 제거한다.
- 새 조건은 다음 흐름을 따른다.

```js
dropFromExpected = expectedRatio - currentRatio;
threshold = trend.ready ? trend.noise : Math.max(0.05, trend.noise);
isCandidate = positiveSteps >= 1 && dropFromExpected >= threshold;
```

### 4. 후보 확정 및 취소
- 후보 이후 관측값이 후보값과 `SCREEN_RECHECK_CONSISTENCY_RATIO` 이내로 비슷하면 확인 횟수를 증가시킨다.
- 2회 이상 확인되면 레벨업을 확정한다.
- 관측값이 후보 생성 전 정상값 근처로 복귀하면 후보를 취소한다.
- 취소 기준은 동적 노이즈 기반 `cancelMargin`을 사용한다.

### 5. 레벨업 확정 후 처리
- 기존 경험치 계산식을 유지한다.

```js
이전 레벨 남은 경험치 + 다음 레벨 현재 진행 경험치
```

- 확정 후 새 레벨 기준으로 추세를 다시 학습하도록 추세 샘플을 초기화한다.
- 레벨업이 포함된 측정 결과는 현재처럼 계산기에 적용하지 못하게 유지한다.

## 기대 동작
- 고레벨에서 작은 상승 후 낮은 값으로 떨어져도 최근 노이즈보다 크면 레벨업 후보가 된다.
- 저레벨에서 초당 20% 이상 상승해도 정상 샘플로 학습되어 상승 자체는 허용된다.
- `30% -> 55% -> 20%`처럼 빠른 구간에서도 상승 추세 뒤 낮은 값이 반복되면 레벨업으로 확정된다.
- 한 프레임만 낮게 튀고 다시 원래 값으로 돌아오면 후보가 취소된다.
