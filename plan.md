# 브라우저 화면 인식 기능 계획 (beta)

## 목표
- 경험치 계산기(`pages/calc/exp.html`)에 화면 캡처 기반 보조 기능을 추가한다.
- 사용자가 게임 화면의 경험치 바 영역을 지정하면, 일정 시간 동안 바 진행률 변화를 측정해서 `1분당 경험치`를 추정한다.
- 추정값은 기존 경험치 계산 탭의 `expPerMinute`와 사이클 최적화 탭의 `cycleExpPerMinute`에 자동 입력할 수 있게 한다.

## 베타 범위
- 우선 Windows Chrome/Edge 최신 버전을 기준으로 검증한다.
- 서버, 백엔드, 업로드 없이 브라우저 안에서만 처리한다.
- `navigator.mediaDevices.getDisplayMedia()`로 사용자가 직접 화면/창/탭을 선택한다.
- 레벨은 OCR로 읽지 않고 사용자가 수동 입력한다.
- 경험치 바 영역은 사용자가 캡처 화면 미리보기에서 직접 드래그로 지정한다.
- 첫 버전은 경험치 바가 같은 레벨 안에서 증가하는 상황만 안정적으로 지원한다.

## 비범위
- 게임 클라이언트 화면을 사용자 동의 없이 자동 선택하지 않는다. 브라우저 보안 정책상 불가능하다.
- 서버 저장, 이미지 업로드, 외부 OCR API 연동은 하지 않는다.
- 레벨업 순간의 경험치 바 리셋을 완전 자동 보정하는 것은 v1에서 제외한다.
- 모바일 브라우저 지원은 제외한다.

## 브라우저 제약
- `getDisplayMedia()`는 HTTPS 또는 localhost 같은 보안 컨텍스트에서만 동작한다. 로컬 검증은 `npx serve .`로 HTTP 서버를 띄우고 브라우저가 허용하는 localhost에서 진행한다.
- 화면 공유 권한은 매번 사용자 조작으로 시작해야 하며, 페이지가 임의로 캡처를 시작할 수 없다.
- 사용자가 창/화면 공유를 중지하면 측정도 즉시 중단해야 한다.
- 캡처 프레임은 `<video>`에 연결하고, 분석은 `<canvas>`에 그린 현재 프레임만 사용한다.

## UX 초안
- 경험치 계산기 안에 새 탭을 추가한다: `화면 인식(beta)`.
- 탭 구성:
  - 안내 문구: 화면 캡처 권한, 서버 전송 없음, 베타 오차 가능성
  - 현재 레벨 입력
  - 측정 시간 선택: 30초, 60초, 120초
  - `화면 선택 시작` 버튼
  - 캡처 미리보기 영역
  - `경험치 바 영역 지정` 상태 표시
  - `측정 시작`, `측정 중지`, `값 적용` 버튼
  - 결과 표시: 시작 진행률, 종료 진행률, 증가 진행률, 추정 1분당 경험치, 신뢰도/주의 문구
- 영역 지정 방식:
  - 미리보기 위에서 드래그하여 사각형 ROI를 만든다.
  - 지정된 ROI는 반투명 박스와 테두리로 표시한다.
  - 다시 드래그하면 ROI를 갱신한다.

## 계산 방식
- `datas/exp.csv`의 현재 레벨 필요 경험치(`expNeed`)를 기준으로 한다.
- 지정 ROI의 가로 방향 픽셀을 분석해 경험치 바가 채워진 비율을 `0~1` 사이 값으로 계산한다.
- 측정 시작 시 `startRatio`, 종료 시 `endRatio`를 저장한다.
- 같은 레벨 안에서 측정했다는 전제로 `deltaRatio = endRatio - startRatio`를 계산한다.
- `gainedExp = expNeed[currentLv] * deltaRatio`
- `expPerMinute = gainedExp / elapsedSeconds * 60`
- `deltaRatio <= 0`이면 레벨업/영역 오류/색상 감지 실패 가능성을 안내하고 자동 적용하지 않는다.

## 바 진행률 감지 초안
- ROI 안의 각 x좌표마다 세로 픽셀의 평균 색상을 구한다.
- 채워진 바는 배경보다 채도 또는 밝기 차이가 크다고 가정한다.
- 초기 구현은 다음 휴리스틱을 사용한다.
  - ROI 왼쪽 10%를 채워진 색상 샘플 후보로 본다.
  - ROI 오른쪽 10%를 비어 있는 배경 샘플 후보로 본다.
  - 각 x좌표 색상이 채워진 샘플에 더 가까우면 filled로 판정한다.
  - 왼쪽부터 연속 filled 구간의 끝을 진행률로 본다.
- 색상 테마/이펙트에 따라 오차가 클 수 있으므로 결과에 `beta` 경고를 표시한다.
- v2 후보: 사용자가 채워진 색/빈 색을 각각 클릭해 샘플링하는 보정 모드 추가.

## 코드 구조 제안
- 새 모듈: `js/calc/screen-exp.js`
  - `startCapture(videoEl)`
  - `stopCapture(stream)`
  - `drawFrameToCanvas(videoEl, canvasEl)`
  - `detectExpRatio(canvasEl, roi)`
  - `calculateExpPerMinute(expNeed, currentLv, startRatio, endRatio, elapsedSeconds)`
- 기존 `js/calc/exp.js` 보완:
  - 현재는 `expNeed`가 모듈 내부 변수라 외부에서 레벨별 필요 경험치를 직접 조회할 수 없다.
  - `getExpNeedForLevel(level)` 같은 작은 export를 추가해 화면 인식 모듈/페이지에서 재사용한다.
- `pages/calc/exp.html` 변경:
  - 탭 버튼과 탭 콘텐츠 추가
  - `screen-exp.js` import
  - 측정 결과를 `expPerMinute`, `cycleExpPerMinute`에 적용하고 기존 저장 로직과 동기화
- `css/style.css` 변경:
  - 캡처 미리보기 컨테이너
  - ROI 오버레이
  - 측정 상태/주의 메시지 스타일

5. 미리보기 위 드래그 ROI 선택 구현
6. 측정 시작/종료 로직 구현
7. 추정 `1분당 경험치`를 기존 입력칸에 적용하는 버튼 추가

10. 대표 화면에서 30초/60초 측정값을 비교해 오차와 안내 문구 조정

## 검증 항목
- `npx serve .` 실행 후 `pages/calc/exp.html`에서 확인한다.
- 화면 공유 시작/중지 버튼이 브라우저 권한 플로우와 충돌하지 않는지 확인한다.
- 공유 중지 시 video track 종료 이벤트로 UI가 중단 상태가 되는지 확인한다.
- ROI를 지정하지 않고 측정 시작하면 명확한 오류를 표시한다.
- 현재 레벨이 비어 있거나 `datas/exp.csv`에 없는 레벨이면 오류를 표시한다.
- 측정 결과 적용 시 `expPerMinute`와 `cycleExpPerMinute`가 같이 갱신되는지 확인한다.
- 기존 경험치 계산/1분당 경험치 계산/사이클 최적화 탭 기능이 깨지지 않는지 확인한다.
- CSV 네트워크 404가 없는지 확인한다.

## 남은 결정 사항
- beta 기능을 기존 경험치 계산기 안의 탭으로 둘지, 별도 계산기 페이지로 분리할지 결정해야 한다.
- 경험치 바 색상 감지가 실제 게임 화면에서 충분한지 확인 후, 필요하면 수동 색상 샘플링을 v1에 포함한다.
- 레벨업이 측정 중 발생했을 때 단순 오류 처리로 둘지, 사용자가 시작/종료 레벨을 입력하는 보정 모드를 넣을지 결정해야 한다.
